## 2. 法语学习系统 (Le Trésor des Mots)

### 2.1 核心机制
- **每日新词**: 每天 5 个新单词
- **记忆曲线**: 基于艾宾浩斯遗忘曲线的间隔重复系统
- **等级系统**: Lvl 1-7,根据掌握程度自动升级
- **星星奖励**: 完成今日所有新词 + 待复习单词 = 100 ⭐(一次性发放)

### 2.2 星星规则详解
```
✅ 星星发放规则:
- 完成条件:掌握所有今日新词 (连续答对 2 次) + 完成所有待复习单词
- 发放时机:完成时弹窗发放 100 ⭐
- 发放方式:一次性奖励,不每题发放

❌ 答对答错的影响:
- 答对:增加掌握进度 (sessionMastery +1),达到 2 次后单词升级
- 答错:重置掌握进度 (sessionMastery = 0),连续 2 次错误降级
- 星星:答对答错都不影响星星,只有完成每日目标才发放 100 ⭐

📊 等级升降规则:
- 升级:连续答对 2 次 → 等级 +1,下次复习日期延后
- 降级:连续答错 2 次 → 等级 -1,下次复习日期提前
- 等级影响:只影响复习频率,不影响星星奖励
```

### 2.3 等级分布说明
```
Lvl 1 (新词): 刚加入,需要频繁复习
Lvl 2-3 (学习中): 正在巩固阶段
Lvl 4-5 (熟练): 大部分单词停留在这里(正常!)
Lvl 6-7 (掌握): 极少数能完全掌握
```

**注意**: Lvl 5 有较多单词是正常现象,说明系统在工作。

### 2.4 复习机制
- **今天到期**: 必须完成的复习单词
- **过期延续**: 如果某天没学习,过期单词自动延续到下次
- **积压还清**: 可通过"还清积压"按钮专项复习

### 2.5 数据同步策略 (v3.8+)
```
1. 优先读取 Supabase 云端数据
2. 云端读取成功后,立即刷新本地缓存
3. 云端不可用时,退回 localStorage 本地缓存
4. 网络恢复后,自动将本地缓存重新同步到云端
```

### 2.6 间隔重复周期
```
Lvl 1 → Lvl 2: 1 天
Lvl 2 → Lvl 3: 2 天
Lvl 3 → Lvl 4: 4 天
Lvl 4 → Lvl 5: 7 天
Lvl 5 → Lvl 6: 15 天
Lvl 6 → Lvl 7: 30 天
Lvl 7 (完全掌握): 60 天
```

## 3. 奖励金系统

### 3.1 奖励规则
| 里程碑 | 奖励 |
|--------|------|
| 每 1000 ⭐ (1个奖杯) | +5 EUR |
| 特殊成就 | 由家长决定 |

### 3.2 自动化 (v3.6+)

- 需填写金额和原因

## 4. 任务管理

### 4.1 每日任务清单
- **来源**: Apple Reminders "童童任务"
- **同步**: 每 30 分钟自动同步
- **原则**: 手机先行 (SSOT)

### 4.2 任务类型
1. 学校作业 (Devoirs)
2. 法语朗读 (10分钟)
3. 词汇积累 (5分钟)
4. 数学应用题 (10分钟)
5. 小提琴练习 (20分钟)

## 5. 技术架构

### 5.1 数据文件
| 文件 | 用途 |
|------|------|
| `french_data.json` | 法语学习数据 |
| `tongtong_stats.json` | 奖励金记录 |
| `jingyi_todo.md` | 每日任务 |

### 5.2 同步机制
- **GitHub Pages**: 静态页面分发
- **云端**: Supabase(主数据源)
- **本地缓存**: 浏览器 localStorage(离线回退)
- **本地后台**: 管理页面仅保留本地,不部署到 GitHub Pages

### 5.3 版本管理
- 每次更新需更新版本号
- 版本号显示在页面底部
- 版本变化时自动清除缓存
